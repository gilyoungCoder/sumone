# Sumone App - 프로젝트 결정 사항

## 프로젝트 개요
- **앱 이름**: Sumone (미국 시장 타겟 커플 앱)
- **원본 참고**: 한국 앱 "썸원" — 연인용 앱, 반려몽 키우기 + 꾸미기 요소 포함
- **타겟 시장**: 미국 (영어)
- **목표**: MVP 빠르게 만들어서 테스트

---

## Q1. 플랫폼
- **결정**: React Native + Expo
- **이유**: 크로스플랫폼, 바이브코딩 최적, Expo로 빠른 개발

## Q2. 핵심 기능 우선순위 (레퍼런스 분석 완료)
- **MVP**: 회원가입/로그인, 커플 연결, D-day, 데일리 질문, 기본 홈(펫), 프로필
- **Phase 2**: 펫 꾸미기, 상점, 재화 시스템, 캘린더, 질문팩, 광고, 구독

## Q3. 백엔드
- **결정**: Supabase (Auth + PostgreSQL + Storage + Realtime)
- **이유**: 2025-26 바이브코딩 트렌드, SQL 기반 유연한 쿼리, 무료 티어 넉넉, 오픈소스

## Q4. 수익 모델
- **결정**: 프리미엄 + 광고 하이브리드
- **상세**:
  - 기본 무료
  - 반려몽(가상 펫) 꾸미기 → 재화 필요 → 광고 시청으로 획득
  - 구독 시 광고 제거 + 프리미엄 기능 해금
  - 원본 썸원의 수익 모델 참고

## Q5. 개발자 프로필
- **경험**: AI 엔지니어, 프로그래밍 언어 기본 이해
- **배경**: 부트캠프 수강 경험 있음
- **개발 방식**: 바이브 코딩 (Claude Code 주도 개발)
- **의미**: 코드 설명은 간결하게, 실행 가능한 결과물 중심으로 진행

## Q6. 우선 목표
- **결정**: MVP (Minimum Viable Product)
- **방향**: 빠른 프로토타이핑 → 테스트 → 반복 개선
